# Contributing to FX Analyzer Pro

Thank you for considering contributing! This document outlines the process.

## Code of Conduct

By participating, you agree to maintain a respectful, inclusive environment.

## How to Contribute

1. **Fork** the repository
2. **Create a branch**: `git checkout -b feature/my-feature`
3. **Make changes** with clear commit messages
4. **Run checks**:
   ```bash
   cd frontend && npm run lint && npm run build
   ```
5. **Push** and open a Pull Request

## Pull Request Guidelines

- Reference the issue number if applicable
- Keep PRs focused — one feature/fix per PR
- Update documentation if changing behavior
- Ensure CI passes (lint, build)

See [README.md](Fx-analyzer/README.md) for local development instructions.
